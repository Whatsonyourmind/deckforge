  <behavior>
    - AdaptiveOverflowHandler.handle(slide, positions, theme, grid, measurer) detects overflow (any region height < measured text height)
    - Overflow cascade step 1: reduce font size by 2pt (minimum 10pt floor), re-measure, re-solve
    - Overflow cascade step 2: if still overflowing at min font, reflow text (aggressive word wrap within narrower width)
    - Overflow cascade step 3: if still overflowing after reflow, split slide into multiple slides with continuation marker
    - Font reduction never goes below 10pt for body or 14pt for headings
    - LayoutEngine.layout_slide(slide, theme, grid) returns LayoutResult with positions
    - LayoutEngine.layout_slide with short content returns overflow=False, no split
    - LayoutEngine.layout_slide with long content (20+ bullet items) triggers adaptive cascade
    - LayoutEngine.layout_presentation(presentation) returns list of LayoutResult for all slides
    - LayoutEngine uses PATTERN_REGISTRY to select pattern per slide type
    - LayoutEngine creates GridSystem from theme.spacing (margins, gutter)
  </behavior>
  <action>
    Create `src/deckforge/layout/overflow.py` with:
    - `AdaptiveOverflowHandler` class
    - Constructor takes `measurer: TextMeasurer`, `min_body_font: int = 10`, `min_heading_font: int = 14`, `font_reduction_step: int = 2`, `max_iterations: int = 5`
    - `detect_overflow(positions: dict[str, ResolvedPosition], measurements: dict[str, BoundingBox]) -> list[str]` -- returns list of region names where measured height exceeds allocated height
    - `handle(slide, positions, measurements, theme, grid, pattern) -> LayoutResult`:
      1. Detect overflow regions
      2. If none: return LayoutResult(overflow=False, positions=positions)
      3. Step 1 - Font reduction: For each overflow region, reduce the font size in a copy of the theme's component style by font_reduction_step. Re-measure with smaller font. Re-generate constraints with new measurements. Re-solve. Check again. Repeat up to max_iterations or until min font reached.
      4. Step 2 - Reflow: If still overflowing, re-measure with max_width reduced by 10% (more aggressive wrapping). Re-solve.
      5. Step 3 - Split: If still overflowing, split the slide. For bullet_list content, split items across two slides. For body_text, split at paragraph boundaries. Return LayoutResult with split_slides populated.
    - `_split_slide(slide, overflow_region, theme) -> list[BaseSlide]` -- creates continuation slides. Each continuation slide has the same slide_type and title (with " (cont.)" suffix), with content divided between slides.
    - `_reduce_font_in_style(style: ComponentStyle, reduction: int, min_size: int) -> ComponentStyle` -- returns new style with reduced font_size, clamped to min_size

    Create `src/deckforge/layout/engine.py` with:
    - `LayoutEngine` class
    - Constructor takes `text_measurer: TextMeasurer`, `theme_registry: ThemeRegistry`
    - `layout_slide(slide: BaseSlide, theme: ResolvedTheme, grid: GridSystem | None = None) -> LayoutResult`:
      1. If grid is None, create GridSystem from theme.spacing
      2. Get pattern from PATTERN_REGISTRY[slide.slide_type]
      3. Get SlideMaster from theme.slide_masters.get(slide.slide_type, default_master)
      4. Measure all text elements using TextMeasurer, producing dict[region_name, BoundingBox]
         - Map elements to regions by type: heading -> "title", subheading -> "subtitle", bullet_list -> "bullets", body_text -> "content", footnote -> "footnote", chart -> "chart_area", table -> "table_area"
      5. Create regions and constraints from pattern
      6. Solve via SlideLayoutSolver
      7. If solve returns None (infeasible), trigger overflow handler directly
      8. If solve succeeds, check for overflow via AdaptiveOverflowHandler.detect_overflow
      9. If overflow detected, run AdaptiveOverflowHandler.handle
      10. Apply resolved positions back to slide elements as Position objects
      11. Return LayoutResult

    - `layout_presentation(presentation: Presentation, brand_kit: BrandKit | None = None) -> list[LayoutResult]`:
      1. Resolve theme via theme_registry.get_theme(presentation.theme, brand_kit or presentation.brand_kit)
      2. Create GridSystem from theme.spacing
      3. For each slide, call layout_slide
      4. Flatten split slides into the results list
      5. Return all LayoutResults

    - `_measure_elements(slide, theme, master, grid) -> dict[str, BoundingBox]`:
      - Walk slide.elements, determine region mapping, measure text using theme's font/size from the SlideMaster
      - For heading elements: use master.regions["title"].font_family and font_size
      - For bullet_list: use master.regions["bullets"].font_family and measure_bullet_list
      - For chart/table/image: return a default BoundingBox (these are sized by available space, not text)

<objective>
Build the Google Slides renderer core: unit conversion layer, batchUpdate request builders for all element types (text, table, image, shape, data viz, finance slides), OAuth credential handler, and the GoogleSlidesRenderer orchestrator.

Purpose: This plan creates the complete request-building pipeline that translates IR + LayoutResult + ResolvedTheme into Google Slides API batchUpdate requests. It parallels the PptxRenderer architecture but targets the Slides API JSON format instead of python-pptx objects.

Output: A GoogleSlidesRenderer class that can create a presentation and populate all slides via batchUpdate. Charts are stubbed as placeholder shapes (Plan 06-02 adds Sheets-backed charts).
</objective>

    **slides_renderer.py:** Create `GoogleSlidesResult` dataclass and `GoogleSlidesRenderer` class:

    ```python
    @dataclass
    class GoogleSlidesResult:
        presentation_id: str
        presentation_url: str  # https://docs.google.com/presentation/d/{id}/edit
        title: str
        slide_count: int
        charts_placeholder: bool  # True until 06-02 wires Sheets charts
    ```

    `GoogleSlidesRenderer`:
    - `__init__(self)` -- no state needed (stateless like PptxRenderer)
    - `render(self, presentation: Presentation, layout_results: list[LayoutResult], theme: ResolvedTheme, credentials) -> GoogleSlidesResult`:
      1. Build Slides API service: `build("slides", "v1", credentials=credentials)`
      2. Create presentation: `service.presentations().create(body={"title": presentation.metadata.title or "DeckForge Presentation"}).execute()`
      3. Get presentation_id from response
      4. Delete the default blank slide that Google creates automatically (get slide ID from presentation response, send DeleteObjectRequest)
      5. Build all requests:
         - For each layout_result, create a SlideRequestBuilder
         - Call `build_create_slide(index)` for each slide
         - Call `build_background(bg_color)` for each slide
         - For finance slides: call `dispatch_finance_slide(ir_slide, theme)` and skip element loop
         - For non-finance slides: iterate elements, call `dispatch_element(element, position, theme)` for each
         - Call `build_speaker_notes(notes)` if ir_slide.speaker_notes exists
      6. Send all requests in a single batchUpdate call (or split into batches of 500 requests if total exceeds 500 -- Google has an undocumented soft limit around 500-1000 requests per batch)
      7. Return GoogleSlidesResult with presentation URL

    Handle errors: catch `googleapiclient.errors.HttpError`, log, re-raise as a DeckForge-specific error. For 429 (rate limit), retry with exponential backoff (max 3 retries, 1s/2s/4s delays).

    **oauth.py:** Create `GoogleOAuthHandler` class:
    - `__init__(self, client_id: str, client_secret: str, redirect_uri: str)` -- stores config
    - `get_authorization_url(self, state: str) -> str` -- builds the Google OAuth consent URL with scopes: `presentations`, `spreadsheets`, `drive.file`. Includes `access_type=offline` and `prompt=consent` to always get refresh token.
    - `exchange_code(self, code: str) -> dict` -- exchanges authorization code for tokens via `https://oauth2.googleapis.com/token`. Returns `{"access_token": ..., "refresh_token": ..., "expires_in": ...}`.
    - `build_credentials(self, access_token: str, refresh_token: str) -> Credentials` -- creates a `google.oauth2.credentials.Credentials` object. Sets token refresh handler so credentials auto-refresh when expired.
    - `refresh_if_needed(self, credentials: Credentials) -> Credentials` -- checks if token expires within 5 minutes, refreshes if so.

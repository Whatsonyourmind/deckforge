Create scripts/bootstrap-db.sh (executable bash script):

```bash
#!/usr/bin/env bash
set -euo pipefail

# DeckForge Database Bootstrap
# Runs Alembic migrations and seeds required data.
# Idempotent: safe to run multiple times.

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
PROJECT_DIR="$(dirname "$SCRIPT_DIR")"

echo "=== DeckForge Database Bootstrap ==="

# Step 1: Run Alembic migrations
echo "[1/3] Running Alembic migrations..."
cd "$PROJECT_DIR"
alembic upgrade head
echo "  Migrations complete."

# Step 2: Seed subscription plans and test data
echo "[2/3] Seeding database..."
python -c "
import sys
sys.path.insert(0, '.')
from src.deckforge.db.engine import get_session_factory
from src.deckforge.db.models.user import User
from src.deckforge.db.models.api_key import ApiKey
from sqlalchemy import select, text
import hashlib, secrets

session_factory = get_session_factory()

with session_factory() as session:
    # Check if test user already exists (idempotent)
    existing = session.execute(
        select(User).where(User.email == 'admin@deckforge.dev')
    ).scalar_one_or_none()

    if existing:
        print('  Seed data already exists (idempotent skip)')
        # Find existing test key
        key = session.execute(
            select(ApiKey).where(ApiKey.user_id == existing.id)
        ).scalar_one_or_none()
        if key:
            print(f'  Test API key: dk_test_{key.id[:12]}...')
    else:
        # Create admin user
        user = User(email='admin@deckforge.dev', name='Admin', tier='starter')
        session.add(user)
        session.flush()

        # Create test API key
        raw_key = 'dk_test_' + secrets.token_hex(24)
        key_hash = hashlib.sha256(raw_key.encode()).hexdigest()
        api_key = ApiKey(
            user_id=user.id,
            key_hash=key_hash,
            prefix=raw_key[:12],
            name='Bootstrap test key',
        )
        session.add(api_key)
        session.commit()
        print(f'  Admin user created: admin@deckforge.dev')
        print(f'  Test API key: {raw_key}')
        print(f'  (Save this key -- it cannot be retrieved later)')

print('  Seeding complete.')
"

echo "[3/3] Verifying database..."
python -c "
import sys
sys.path.insert(0, '.')
from src.deckforge.db.engine import get_session_factory

session_factory = get_session_factory()
with session_factory() as session:

    count = result.scalar()
    print(f'  Tables: {count}')
    assert count and count > 0, 'No tables found -- migrations may have failed'
print('  Database verification passed.')
"

echo ""
echo "=== Bootstrap Complete ==="
echo "Run: curl http://localhost:8000/v1/health"
```

# DeckForge

API-first presentation generation platform. 32 slide types, 15 themes, native PPTX + Google Slides output, AI content generation, finance vertical.

## Tech Stack

- **API**: Python 3.12, FastAPI, Pydantic v2, uvicorn
- **Database**: PostgreSQL (prod) / SQLite (dev), SQLAlchemy 2.0, Alembic
- **Queue**: Redis + ARQ (async workers)
- **Storage**: MinIO (dev) / Cloudflare R2 (prod), S3-compatible
- **Rendering**: python-pptx, google-api-python-client, Plotly (static charts), Kaleido
- **SDK**: TypeScript, tsup, vitest (in sdk/ directory)
- **Billing**: Stripe (subscriptions), x402 (USDC per-call for agents)
- **Auth**: Unkey (prod) / DB-based SHA-256 (dev), API key prefix dk_test_ / dk_live_
- **Layout**: kiwisolver constraint solver, 12-column grid, Pillow text measurement
- **Themes**: YAML definitions in src/deckforge/themes/data/
- **Content**: Model-agnostic LLM orchestration (Claude, OpenAI, Gemini, Ollama)

## Project Structure

src/deckforge/
  ir/           # Pydantic IR models (32 slide types, element types, chart types)
  api/          # FastAPI routes, middleware (auth, rate limit, credits, x402)
  db/           # SQLAlchemy models, repositories, Alembic migrations
  rendering/    # PPTX renderer, Google Slides renderer, chart renderers, thumbnails
  layout/       # Constraint-based layout solver, grid system, patterns
  themes/       # 15 YAML themes, resolver, brand kit merger
  content/      # NL-to-IR pipeline (intent -> outline -> expand -> refine)
  llm/          # LLM adapters (Claude, OpenAI, Gemini, Ollama), router
  charts/       # Chart recommender
  finance/      # Financial formatter, conditional formatting, data ingestion
  qa/           # 5-pass QA pipeline, auto-fix engine, scorer
  billing/      # Stripe client, credit system, tier definitions, x402 config
  services/     # Deck operations, slide type registry, cost estimator
  workers/      # ARQ worker settings, task definitions, webhooks
  mcp/          # MCP server (6 tools for AI agent integration)
sdk/            # TypeScript SDK (@deckforge/sdk)
demos/          # 5 demo deck IRs (McKinsey, PE, startup, board, product)
alembic/        # Database migrations

## Key Commands

```bash
# Local development
docker compose up -d                    # Start all services
bash scripts/bootstrap-db.sh            # Initialize database
pytest                                  # Run Python tests (800+)
cd sdk && npm test                      # Run SDK tests

# Building
cd sdk && npm run build                 # Build TypeScript SDK

# Deployment
fly deploy                              # Deploy to Fly.io
fly secrets set KEY=value               # Set production secrets
```

## Architecture Patterns

- **IR Schema**: Pydantic discriminated unions on slide_type/type/chart_type fields. All 32 slide types in ir/slides/, elements in ir/elements/, charts in ir/charts/
- **Renderer Registry**: ELEMENT_RENDERERS dict maps element.type -> renderer instance. Finance slides use render_finance_slide() with early return before element loop
- **Layout Patterns**: PATTERN_REGISTRY maps SlideType -> pattern class. GenericPattern as fallback. Adaptive overflow: font reduce -> reflow -> split
- **Repository Pattern**: Singleton instances, session-as-argument. Located in db/repositories/
- **Config**: Pydantic Settings with DECKFORGE_ prefix, loaded from .env
- **Workers**: ARQ with Redis, ContentWorkerSettings and RenderWorkerSettings, worker context dict pattern
- **Auth**: AuthContext dataclass with .id property, dual auth (Unkey prod / DB dev)
- **Billing**: Credit reservation before render, deduction on success, release on failure. Enterprise allows overage.

## Coding Conventions

- Python: Type hints everywhere, Pydantic models for all schemas, async endpoints
- Imports: Local imports in worker tasks to avoid circular dependencies
- Tests: pytest with fixtures, located alongside source or in tests/
- FastAPI: Annotated dependency injection (DbSession, RedisClient, CurrentApiKey)
- Error handling: HTTPException with descriptive messages, 422 for validation
- Config: All settings via DECKFORGE_ environment variables
```

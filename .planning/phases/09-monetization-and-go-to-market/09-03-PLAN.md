    1. Create src/deckforge/db/repositories/analytics.py (AnalyticsRepository):
       - `get_overview(db, days: int = 30) -> dict`:
         - Total API calls (COUNT usage_records in period)
         - Total Stripe revenue (SUM credits * overage_rate from usage_records where tier != "x402")
         - Total x402 revenue (SUM amount_usd from payment_events in period)
         - Active consumers (COUNT DISTINCT api_key_id from usage_records)
       - `get_endpoint_breakdown(db, days: int = 30) -> list[dict]`:
         - GROUP BY endpoint, COUNT calls, SUM credits
       - `get_top_consumers(db, days: int = 30, limit: int = 20) -> list[dict]`:
         - GROUP BY api_key_id, COUNT calls, SUM credits, ORDER BY calls DESC
       - `get_revenue_timeseries(db, days: int = 30, granularity: str = "daily") -> list[dict]`:
         - For each day/week/month: stripe_revenue, x402_revenue, total_revenue
         - Uses date_trunc for grouping
       All queries use SQLAlchemy select() with func.count/func.sum/func.date_trunc.
       Use existing DbSession pattern. Return dicts, not models.

    2. Create src/deckforge/api/schemas/analytics_schemas.py:
       - OverviewResponse: total_calls (int), stripe_revenue_usd (float), x402_revenue_usd (float), total_revenue_usd (float), active_consumers (int), period_days (int)
       - EndpointBreakdown: endpoint (str), calls (int), credits_used (int)
       - ConsumerBreakdown: api_key_id (str), calls (int), credits_used (int), tier (str)
       - RevenueDatapoint: date (str), stripe_revenue_usd (float), x402_revenue_usd (float), total_revenue_usd (float)
       - AnalyticsResponse: overview (OverviewResponse), endpoints (list[EndpointBreakdown]), top_consumers (list[ConsumerBreakdown]), revenue_trend (list[RevenueDatapoint])

       - GET /v1/analytics/consumers?days=30&limit=20 -- returns list[ConsumerBreakdown]

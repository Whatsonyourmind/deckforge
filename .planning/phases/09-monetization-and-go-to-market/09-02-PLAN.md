    3. Create src/deckforge/mcp/tools.py with 6 tool implementation functions:
       - `render_presentation(ir_json: str, theme: str = "corporate_navy", output_format: str = "pptx") -> dict`:
         Parse ir_json, call render_pipeline internally, save to S3, return {"download_url": ..., "slide_count": ..., "quality_score": ...}
       - `generate_presentation(prompt: str, slide_count: int = 10, theme: str = "corporate_navy") -> dict`:
         Call content pipeline to generate IR from prompt, then render, return {"download_url": ..., "slide_count": ..., "title": ...}
       - `list_themes() -> list[dict]`:
         Call ThemeRegistry.list_themes(), return list of {name, description, colors: {primary, accent, background}}
       - `list_slide_types(category: str | None = None) -> list[dict]`:
         Call SlideTypeRegistry.get_all() or get_by_category(), return list of {slide_type, description, category}
       - `estimate_cost(ir_json: str) -> dict`:
         Parse IR, call CostEstimator, return {"credits": ..., "x402_usd": ...}
       - `get_pricing() -> dict`:
         Return tiers + x402 per-call pricing (same data as GET /v1/pricing)

       Each function should be a standalone async function that imports what it needs locally (lazy imports to avoid circular deps). Functions accept simple types (str, int) and return dicts -- MCP handles JSON serialization.

       - Import FastMCP from mcp.server.fastmcp

       - Add descriptive docstrings to each tool (agents read these for discovery)
       - Add `if __name__ == "__main__": mcp.run(transport="stdio")` for Claude Desktop
       - Also expose `mcp.run(transport="streamable-http")` option via CLI arg for production

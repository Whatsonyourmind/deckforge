       - `NumberFormat` enum: `CURRENCY`, `CURRENCY_FULL`, `PERCENTAGE`, `MULTIPLE`, `BASIS_POINTS`, `RATIO`, `INTEGER`, `PLAIN`.
       - `FinancialFormatter` class with static methods:
         - `currency(value: float, symbol: str = "$", precision: int = 1, compact: bool = True, negative_parens: bool = True) -> str`:
           If compact, use magnitude suffixes: T (>=1e12), B (>=1e9), M (>=1e6), K (>=1e3).
           Negative values: use parentheses if `negative_parens=True`, else minus sign.
           For non-compact: `f"{symbol}{abs(value):,.{precision}f}"` with parens/minus for negative.
         - `percentage(value: float, precision: int = 1, is_decimal: bool = True) -> str`:
           If is_decimal, multiply by 100. Format to precision decimals + "%".
         - `multiple(value: float, precision: int = 1) -> str`:
           Format to precision decimals + "x".
         - `basis_points(value: float, is_decimal: bool = True) -> str`:
           If is_decimal, multiply by 10000. Round to integer + "bps".
         - `ratio(value: float, precision: int = 2) -> str`:
           Format to precision decimals, no suffix.
         - `auto_format(value: float, format_type: str | NumberFormat) -> str`:
           Dispatch to the correct method based on format_type string/enum.

    3. Create `src/deckforge/finance/conditional.py`:
       - `ConditionalFormatter` class with static methods:
         - `pos_neg_color(value: float, theme: ResolvedTheme) -> str`:
           Returns theme.colors.positive for >0, theme.colors.negative for <0, theme.colors.text_muted for 0.
         - `pos_neg_text_color(value: float, theme: ResolvedTheme) -> str`:
           Same logic but for text color (returns darker variants).
         - `median_highlight(value: float, median: float, theme: ResolvedTheme) -> str`:
           Returns lightened positive color for above median, lightened negative for below, surface for equal.
           `_lighten(hex_color, factor=0.85)`: Blends toward white by factor (0.85 = 85% white + 15% color).
         - `heatmap_gradient(value: float, min_val: float, max_val: float, theme: ResolvedTheme) -> str`:
           Linear interpolation between negative color (low) and positive color (high).
           `_interpolate_color(color1, color2, ratio)`: RGB component-wise interpolation.
         - `traffic_light(status: str) -> str`:
           Returns fixed colors: "green" -> "#27AE60", "yellow" -> "#F39C12", "red" -> "#E74C3C".
       - Helper `_hex_to_rgb_tuple(hex_color: str) -> tuple[int, int, int]` for color math.
       - Helper `_rgb_tuple_to_hex(r: int, g: int, b: int) -> str` for color output.

    1. Add to pyproject.toml [project.dependencies]: `"pandas>=2.2"`, `"openpyxl>=3.1"`.

    2. Create `src/deckforge/finance/ingestion.py`:

       - `ColumnType` enum: `TEXT`, `CURRENCY`, `PERCENTAGE`, `MULTIPLE`, `BASIS_POINTS`, `NUMERIC`, `DATE`.

       - `ColumnMapping` dataclass: `name: str`, `column_type: ColumnType`, `format_type: NumberFormat | None`, `role: str` (one of: "label", "metric", "date", "ignore").

       - `DataMapping` dataclass: `detected_slide_type: str`, `columns: list[ColumnMapping]`, `confidence: float`, `notes: str`.

       - `ingest_tabular_data(source: str | bytes | BinaryIO, file_type: str = "auto") -> pd.DataFrame`:
         - Auto-detect file_type from extension if "auto" and source is a string path.
         - For "csv"/"tsv": `pd.read_csv(source, sep="\t" if tsv)`.
         - For "xlsx": `pd.read_excel(source, engine="openpyxl")`.
         - Raise `ValueError` for unsupported types.
         - Strip whitespace from column names.
         - Return the DataFrame.

       - `detect_column_types(df: pd.DataFrame) -> list[ColumnMapping]`:
         Heuristics applied per column:
         1. Check header name for keywords: "company"/"name"/"ticker" -> TEXT/label, "EV/EBITDA"/"P/E"/"multiple" -> MULTIPLE/metric, "$"/"revenue"/"market cap"/"value"/"price" -> CURRENCY/metric, "%"/"margin"/"growth"/"yield"/"rate" -> PERCENTAGE/metric, "date"/"year" -> DATE/date.
         2. If header heuristic inconclusive, check values: all strings -> TEXT, 0-1 float range -> PERCENTAGE, values ending in "x" -> MULTIPLE, values with "$" -> CURRENCY, all numeric -> NUMERIC.
         3. First text column is role="label", all others role="metric" unless overridden.

       - `auto_detect_slide_type(df: pd.DataFrame) -> DataMapping`:
         Decision logic:
         1. If exactly 2 text columns (row_header, col_header) + 2D numeric grid -> "sensitivity_table".
         2. If first column is text + remaining are numeric with financial headers -> "comp_table".
         3. If columns include "source"/"from" and "target"/"to" and "value" -> sankey data (not a finance slide, return "chart:sankey").
         4. If columns include "start" and "end"/"finish" -> gantt data (return "chart:gantt").
         5. If columns include "low"/"min" and "high"/"max" -> "chart:football_field".
         6. Default: "comp_table" with lower confidence.
         Return `DataMapping` with slide type, column mappings, and confidence.


<objective>
Build native editable chart renderers for all python-pptx-supported chart types. Charts rendered natively are editable in PowerPoint -- users can click them and modify the underlying data. This plan creates the chart renderer registry and implementations for bar, line, area, pie, donut, scatter, bubble, combo, and radar chart types.

Purpose: Native editable charts are a key differentiator -- presentations feel like they were hand-built in PowerPoint, not exported from a tool. PPTX-02 requires 10 native chart types.
Output: `src/deckforge/rendering/chart_renderers/` package with renderers for all native chart types, integrated with the element renderer for ChartElement.
</objective>

    values: list[float | int]

    title: str | None = None

    values: list[float | int]

    title: str | None = None

    1. `chart_renderers/base.py`: Abstract base class:
       ```python
       class BaseChartRenderer(ABC):
           @abstractmethod
           def render(self, slide, chart_data, position: Position, theme: ResolvedTheme) -> None:
               """Render a chart onto a python-pptx slide at the given position."""

           def _apply_theme_formatting(self, chart, theme: ResolvedTheme) -> None:
               """Apply theme colors, fonts, and styling to a chart."""
       ```
       The base class provides shared `_apply_theme_formatting` that:
       - Sets chart series colors from theme.chart_colors (cycling if more series than colors)
       - Sets tick label font to theme.typography.body_family, caption scale size
       - Sets legend font to body_family, caption size
       - Positions legend at bottom (XL_LEGEND_POSITION.BOTTOM)
       - Sets chart title font to heading_family, h3 scale if title exists

    2. `chart_renderers/category.py`: Category-based charts (most common):
       - `BarChartRenderer`: Handles bar, stacked_bar, grouped_bar, horizontal_bar
         - Map chart_type to XL_CHART_TYPE: bar->COLUMN_CLUSTERED, stacked_bar->COLUMN_STACKED, grouped_bar->COLUMN_CLUSTERED, horizontal_bar->BAR_CLUSTERED
         - Create CategoryChartData, set categories from chart_data.categories
         - Add each ChartDataSeries via chart_data.add_series(series.name, series.values)
         - Call shapes.add_chart(chart_type, left, top, width, height, chart_data)
         - If show_values: enable data labels on the plot
         - Apply theme formatting
       - `LineChartRenderer`: Handles line, multi_line
         - Map: line->LINE_MARKERS (or LINE if show_markers=False), multi_line->LINE_MARKERS
         - Same CategoryChartData pattern
         - Apply theme formatting (line color = chart_colors per series)
       - `AreaChartRenderer`: Handles area, stacked_area
         - Map: area->AREA, stacked_area->AREA_STACKED
         - Same CategoryChartData pattern

    3. `chart_renderers/proportional.py`: Pie and donut:
       - `PieChartRenderer`:
         - Create CategoryChartData with chart_data.labels as categories
         - Add single series "Values" with chart_data.values
         - Create PIE chart
         - If show_percentages: enable data labels with percentage format
         - Color each slice using theme.chart_colors
       - `DonutChartRenderer`:
         - Same as pie but DOUGHNUT chart type
         - Set hole size from chart_data.inner_radius via lxml: access chart._chartSpace, find doughnut plot, set holeSize attribute to int(inner_radius * 100)
